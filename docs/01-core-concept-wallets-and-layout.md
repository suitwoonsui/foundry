# Core concept, wallets, and page layout

Working name **The Community Foundry**, per-project Sui wallets, supporter benefits, narrative case, example UI, and value to you. Next: [02-pillars-paths-and-funding-page.md](02-pillars-paths-and-funding-page.md).

---

Core Concept:

Create a page of our project where the community can donate to the project. Providing donation can provide benefits to those who donate. This could be NFTs, reduced prices on this, token rewards, etc. depending on the project. The projects can include apps and games we are building or marketing campaigns. Each project would likely have it's own Sui Wallet, making it easier to track and provide details on targets and current holdings as well as transactions.

Working Name
The Community Foundry
(A hub where supporters can directly fuel specific projects and in return receive recognition, access, rewards, or stake in the ecosystem.)

Core Concept
Create a dedicated project funding page where community members can contribute to the development of individual products, games, tools, or marketing pushes. Instead of one single donation pool, each project has its own dedicated Sui wallet, making funding transparent and traceable.
This lets the community:
Support what they truly believe in


Watch progress happen in real time


Receive meaningful benefits tied directly to the project they funded


This also gives you a clear ledger of:
Project runway


Resource allocation


Development milestones per project



How Funding Works
Each project has its own Sui wallet.


Example:

 wallet_recon_tactics.sui

 wallet_crypto_tracker.sui

 wallet_marketing_push_march2026.sui


The funding page displays:


Project mission and scope


Funding goal


Current wallet balance (auto-refreshed or API pulled)


Transaction transparency log


Development progress timeline


Supporters donate directly to the wallet of the project they choose.



Supporter Benefits
Contribution ≠ charity.
Supporters gain something tangible based on the project type and contribution tier.
Examples:
Early Access


Reduced rates or lifetime discount


Exclusive NFT badges or identity markers


Token allotments or boosted allocations


Path-specific role or title in Discord/Telegram


Voting influence on feature roadmap decisions


First priority whitelist access


Benefits scale with contribution tiers:
Tier 1 – Early access & badge


Tier 2 – Discounts or early token allocation


Tier 3 – Governance-level benefits, lore integration, unique perks


This is basically Kickstarter + Web3 transparency + identity + community loyalty + actual upside.

Why This Works (Narrative Logic)
People support things they feel ownership in.
By letting the community choose which project to fuel, you:
Signal that their voice shapes direction


Avoid ever looking like you're just fundraising "into the void"


Build community as stakeholders, not just spectators


And the blockchain wallet transparency ensures:
Trust


Credibility


Shared accountability


Zero "black box funding"


This is collective building instead of donor guilt.
