# Ecosystem roadmap (phases 1–11)

High-level rollout from project hub through platform expansion. Previous: [03-path-naming-options.md](03-path-naming-options.md). Next: [05-phase-1-site-ia-and-ux.md](05-phase-1-site-ia-and-ux.md).

---

High-Level Ecosystem Roadmap
Phase 1 — Project Visibility
Create the Project Hub page


Display all active + upcoming projects


Show purpose, type, status (but no funding yet)


Outcome:
Community clearly sees what we are building.

Phase 2 — Donations / Funding Activation
Add per-project Sui wallets


Show funding goals + progress bars


Enable direct contribution


Outcome:
Community can fuel what matters to them.

Phase 3 — Onboarding & Introduction
Create a simple "How to Get Involved" guide


Pin it in Telegram + link from site


Explain participation options & weekly rhythm


Outcome:
New members understand how to enter the movement.

Phase 4 — Weekly Progress Cycle
Weekly project updates during weekly Spaces


Recap shared in Telegram


Each project shows milestone movement


Outcome:
Community sees progress unfolding in real time.
Momentum becomes felt, not assumed.

Phase 5 — Manual Rewards / Contribution Benefits
Contributors receive perks related to the project they supported


Examples: early access, reduced pricing, whitelist priority


No automation yet — low friction, high meaning


Outcome:
Contribution feels valuable and personal.

Phase 6 — Identity Paths (Contribution Types)
Define the six paths under **Six Pillars of the Foundry** (Heralds, Forge, Vault, Diplomats, Orators, Council)


This is still cultural / manual — not automated yet


People choose how they want to participate


Outcome:
Ownership shifts from money → meaning → identity.

Phase 7 — Project Submission System
Community can propose new project ideas


Submission form + core team review


Approved ideas enter Phase 1


Outcome:
Community becomes co-creator, not just participant.

Phase 8 — Dynamic Badge / Identity NFT System
Introduce evolving contributor badges (soulbound identity NFTs)


Badges reflect path + contribution depth


Still minimal automation at first


Outcome:
Identity becomes visible, persistent, and story-driven.

Phase 9 — Reward & Access Automation
System begins automatically:


Recognizing contributions


Updating badge state


Granting perks


The logic layer becomes independent from UI (Telegram, website, etc.)


Outcome:
The ecosystem becomes self-maintaining without losing culture.

Phase 10 — Telegram Integration Layer
Telegram bot reflects identity + access levels (optional)


Path chats + contributor groups unlock automatically


Outcome:
The social layer mirrors the identity ecosystem.

Phase 11 — Platform Expansion
Other communities/projects can use your model


Your system evolves into a Community-Powered Accelerator


Outcome:
Your ecosystem becomes a launch engine.
