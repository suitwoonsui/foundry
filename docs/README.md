# Community Foundry — concept documents

Initial concept for **The Community Foundry**: per-project Sui wallets, transparent funding, supporter benefits, **Six Pillars of the Foundry** (umbrella) and **paths** (how people contribute and level up), rollout phases, and Phase 1 site IA/UX detail.

| File | Contents |
|------|----------|
| [01-core-concept-wallets-and-layout.md](01-core-concept-wallets-and-layout.md) | Core idea, working name, how funding works, benefits, narrative, example layout, value to you |
| [02-pillars-paths-and-funding-page.md](02-pillars-paths-and-funding-page.md) | Culture (belonging vs pay-to-influence), pillars + paths, recognition levels, Vault as one path, funding page as entry gate |
| [03-path-naming-options.md](03-path-naming-options.md) | Alternate path names per contribution style |
| [04-ecosystem-roadmap-phases.md](04-ecosystem-roadmap-phases.md) | Phases 1–11 (visibility → platform expansion) |
| [05-phase-1-site-ia-and-ux.md](05-phase-1-site-ia-and-ux.md) | Phase 1 only: routes, categories, filters, search, cards, detail page, design system, accessibility |

## Vocabulary

- **Six Pillars of the Foundry** — use for the umbrella / program framing only.
- **Path** — day-to-day language for a contribution lane and progression (levels, onboarding, site `/paths`).
